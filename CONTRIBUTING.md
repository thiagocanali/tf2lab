# Como Contribuir

Obrigado por considerar contribuir com o TF2Lab! 🎉

## Código de Conduta

Ao participar deste projeto, você concorda com nosso [Código de Conduta](CODE_OF_CONDUCT.md).

## Como começar

1. Faça um fork do repositório
2. Crie uma branch para sua feature:
   git checkout -b feature/nome-da-feature
3. Faça suas alterações
4. Teste localmente (npm run dev)
5. Commit com mensagem clara (seguindo Conventional Commits)
6. Abra um Pull Request

## O que aceitamos

Bug fixes
Novas features (discuta antes em uma Issue)
Melhorias de documentação
Novos componentes UI (seguindo o Design System)
Integrações com APIs públicas do TF2/logs.tf

## Estrutura do Projeto

features/ → Código organizado por domínio (analytics, competition, etc.)
components/ui/ → Componentes genéricos reutilizáveis
shared/ → Utilitários, tipos, constantes
docs/ → Documentação técnica e de produto

## Pull Requests

Título claro e descritivo
Descreva o que mudou e por quê
Adicione testes quando aplicável
Mantenha o código limpo e tipado

Qualquer dúvida, abra uma Issue ou converse com o mantenedor.
Vamos construir algo incrível juntos! 🚀
